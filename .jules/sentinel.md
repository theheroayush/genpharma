## 2024-03-10 - XSS via Unescaped Patient Data in Document Write\n**Vulnerability:** Found unescaped user data (`prescription?.patientName` and `prescription?.patientInitials`) being inserted directly into `document.write()` in `PouchScheduleView.tsx`.\n**Learning:** This is a classic DOM-based XSS vulnerability. If a malicious user registers with a name containing `<script>alert(1)</script>`, it will be executed when the pharmacist prints the labels.\n**Prevention:** Always escape user input when generating HTML via string concatenation, or use a proper templating engine / React components for generating printable views.
## 2024-03-15 - CSS Injection via dangerouslySetInnerHTML in ChartStyle
**Vulnerability:** Found unsanitized variables (`id`, `key`, `color`) being injected into a `<style>` block using `dangerouslySetInnerHTML` in `src/components/ui/chart.tsx`.
**Learning:** Even internal UI components generated by tools like Shadcn can introduce vulnerabilities if they construct CSS dynamically from configuration that might eventually be user-controlled. Specifically, injecting values without escaping quotes, brackets, or angle brackets allows an attacker to break out of the CSS rule and inject arbitrary HTML/scripts.
**Prevention:** Always sanitize inputs used in dynamic CSS blocks. For identifiers, restrict to `[^\w-]`. For values, strip structural characters like `['";{}<>]`.
