## 2024-05-18 - Privilege Escalation via User Metadata
**Vulnerability:** A critical vulnerability existed in the database trigger `handle_new_user` where it auto-approved the `role` provided in `NEW.raw_user_meta_data` without explicitly sanitizing it. An attacker could manually intercept or craft a request to `supabase.auth.signUp` setting `role: 'admin'` to grant themselves elevated auto-approved privileges.
**Learning:** `raw_user_meta_data` is client-controllable input during signup. While it can be used for *requested* roles that require manual admin approval (like `pharmacist`), it must never be blindly trusted for auto-approved or critical roles (like `admin`). Complete removal of the client payload breaks role-request workflows, so sanitization in the trigger is the correct approach.
**Prevention:** Always sanitize client-provided role metadata in backend functions/triggers handling user registration. Explicitly downgrade or reject attempts to request critical roles (`'admin'`), and ensure requested non-default roles (e.g., `'pharmacist'`) are marked with `approved = false` to enforce manual administrator review.
