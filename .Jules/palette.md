## 2024-05-24 - Accessibility gaps in custom authentication inputs
**Learning:** React components (like `Input` or manual `<input>` elements wrapped with custom actions) often have visually descriptive icons for actions like "Toggle password visibility" but lack proper ARIA labels. Because `aria-label` is not universally enforced by default on naked buttons wrapping standard icon spans (`<span className="material-symbols-outlined">visibility</span>`), these critical security interactions are completely opaque to screen reader users.
**Action:** When auditing or implementing authentication forms (or any forms with inline actions), specifically check for icon-only buttons (`size="icon"` or manual wrappers) and ensure they use a dynamic `aria-label` that reflects the current state (e.g., "Show password" / "Hide password").
